## 1. V0.2 目标

V0.1 已跑通“小程序登录、兑换、获得小鸟、每日鸟签、AI 对话、分享串门、后台管理”的主链路。

V0.2 的目标不是增加更多入口，而是把体验从“功能可用”升级为“用户愿意每天回来看看小鸟”。

核心升级：

1. 小鸟从资料卡变成有状态、有反应的陪伴对象。
2. 每日鸟签从单次查看变成一天的情绪仪式。
3. AI 聊天从工具入口变成影响小鸟状态的互动。
4. 串门从普通分享变成轻社交惊喜。
5. UI 从工程骨架升级为高保真、可爱、温柔的产品界面。

## 2. 产品判断

当前问题：

1. 玩法不够：用户每天回来后能做的事太少。
2. 宠物不够可爱：小鸟还像鸟类资料，不像“我的小鸟”。
3. UI 高保真不足：首页、聊天页、鸟签页还缺情绪、层级和品牌质感。

V0.2 解决方式：

```text
每日心情 -> 今日鸟签 -> 陪小鸟说话 -> 小鸟状态变好 -> 得到一片羽毛/一张小纸条
```

这条循环轻，不像硬核游戏；温柔，符合 JOYIBIRD；可落地，不需要复杂经济系统。

## 3. 核心玩法一句话

每天来看看你的小鸟，它会带来一张鸟签。你陪它说几句话，它的心情会好一点，也会更熟悉你一点。

## 4. V0.2 核心循环

### 4.1 每日主循环

1. 用户进入小鸟主页。
2. 首页展示小鸟今日心情 + 小鸟今天发生了什么（如“昨晚做了一个梦”）。
3. 用户查看今日鸟签。
4. 小鸟根据鸟签给出一句反应。
5.（若有跨天记忆）小鸟自然地提起用户昨天说过的事。
6. 用户和小鸟说话。
7. 小鸟心情发生变化。
8.（偶发）小鸟叼来一个小东西（树叶/小石头/亮片）。
9. 系统记录今日陪伴完成。
10. 用户获得一片羽毛或一张小纸条。

### 4.2 次日回访循环

1. 小鸟展示新的今日心情。
2. 如果昨天完成陪伴，小鸟会记得。
3. 如果昨天没有来，小鸟不责备，只轻轻表达等待。
4. 连续陪伴提升熟悉度。

示例文案：

```text
昨天你来过以后，它睡得很安稳。
它今天在窗边站了一会儿，好像在等你。
没关系，你现在来了，它已经很开心。
```

### 4.3 随机事件与惊喜

每天的主体循环是稳定的，但需要低概率的意外来制造“今天会有什么不同”的期待。

V0.2 建议先做 4 种随机事件，每日触发概率 15%-25%（即平均 4-7 天发生一次）：

| 事件 | 触发概率 | 效果 | 展示 |
|---|---|---|---|
| 小鸟叼来东西 | 8% | 获得一片特别的羽毛/一片叶子/一颗小石头 | 首页弹窗：“它今天叼回来一片亮亮的叶子，轻轻放在你手边。” |
| 小鸟做了一个梦 | 5% | 生成一段 2-3 句的梦境小故事 | 聊天页小鸟主动讲述：“我昨晚做了一个梦。梦见窗外有一大片向日葵。” |
| 朋友的小鸟来过 | 已由串门触发 | 首页温柔提醒 | 见 6.5 |
| 小鸟发现了什么 | 5% | 小鸟在窗边看到了什么有趣的事 | 首页状态：“它今天在窗边站了很久。外面有一只蝴蝶飞过去了。” |

规则：

1. 随机事件在用户进入首页时 roll，当天固定不再变。
2. 事件不阻塞主流程，作为首页或聊天页的额外内容。
3. 不做负面事件（如噩梦、生病、丢了东西）。
4. “叼来东西”收集到的东西可以在首页轻量展示，慢慢形成一个“小鸟的收藏”。

## 5. 三个核心系统

### 5.1 小鸟心情

小鸟每天有一个可见心情。心情不是数值压力，而是情绪陪伴入口。

V0.2 建议先做 6 个心情：

| mood | 展示名 | 首页一句话 | 小鸟今天发生了什么（叙事层） | 触发倾向 |
|---|---|---|---|---|
| sleepy | 困困 | 它今天把脑袋埋进羽毛里。 | 昨晚窗外的路灯太亮了，它啄了半晚上的窗帘，现在困得不行。 | 用户久未互动、早晨 |
| bright | 亮亮 | 它好像发现了一点好事。 | 早上窗台上来了一只蜜蜂，它好奇地看了很久，现在心情很好。 | 完成鸟签、完成聊天 |
| quiet | 安静 | 它今天话不多，但一直在。 | 今天外面下着小雨，它就在窗边静静地听雨声。 | 默认、低能量 |
| waiting | 等你 | 它在窗边站了一会儿。 | 它今天在窗台上走了好几个来回，每次有动静就抬头看一眼。 | 昨天未互动 |
| messy | 乱乱 | 它的羽毛有点翘。 | 它刚才扑腾着整理羽毛，结果越整理越乱，干脆不管了。 | 用户表达焦虑/混乱 |
| cozy | 暖暖 | 它轻轻靠近了一点。 | 它把自己缩成一个圆圆的毛球，看起来特别暖和。 | 聊天后、连续陪伴 |

规则：

1. 每日首次进入生成初始心情。
2. 查看鸟签后允许心情变化一次。
3. 聊天后允许心情变化一次。
4. 不做负面惩罚，不做“生病”“饿了”等压力型状态。

### 5.2 熟悉度与羽毛

熟悉度是轻养成，不做复杂等级经济。

展示方式：

```text
熟悉度：刚认识 / 会等你 / 认得你的脚步声 / 把你当窝边的人
羽毛：今天 +1
```

V0.2 规则：

1. 每日查看鸟签：`feather_count +1`。
2. 每日首次完成聊天：`bond_score +1`。
3. 连续陪伴：额外显示温柔反馈，不强制奖励。
4. 熟悉度只影响文案、称呼、开场白，不影响核心权限。

熟悉度阶段：

| bond_level | 名称 | 分数 | 小鸟表现 | 行为差异 |
|---|---|---:|---|---|
| 1 | 刚认识 | 0-2 | 礼貌、轻轻靠近 | 小鸟只是看着你，偶尔靠近。开场白较正式。 |
| 2 | 会等你 | 3-6 | 记得你昨天来过 | 进入首页时小鸟会“跳一下”表示高兴。开场白提及“昨天”。 |
| 3 | 认得你的脚步声 | 7-14 | 主动开场更自然 | 小鸟会主动跟你分享它今天发现了什么。聊天中偶尔引用之前你说过的话。 |
| 4 | 把你当窝边的人 | 15+ | 更亲密、更像专属陪伴 | 小鸟偶尔会主动开启聊天（不用等用户点击按钮）。在记忆中引用用户很久以前说过的话，更像老朋友。 |

核心原则：bond_level 提升的不是权限，而是小鸟的**主动性**——级别越高，小鸟越主动接近、分享、回忆。这让 bond_score 不只是数字，而是用户能感知到的态度变化。

### 5.3 小纸条

小纸条是每日闭环奖励。它比积分更可爱，也适合分享。

触发：

1. 用户当天查看鸟签并完成一次聊天。
2. 小鸟生成一张今日小纸条。
3. 小纸条可以留在首页，也可以分享。

小纸条格式：

```text
今天它想留给你：
“不用一下子变好。先把水喝了，也算往前走。”
```

生成规则：

1. **模板优先策略**：小纸条主体由模板 + 变量（鸟签名、心情名、用户名字、小鸟名字）组合生成，保证品质稳定。
2. AI 作为**润色层**：在模板生成基础上，AI 可以对文案做 1-2 字的微调使其更自然。AI 不可用时不润色，模板原文直接展示。
3. 同一天固定，不重复生成。
4. 模板库至少准备 20 条，覆盖不同的心情组合（如“quiet + 完成陪伴”“bright + 周五”等），避免用户感觉到重复。

设计理由：避免 AI 生成和模板生成的品质断崖式落差——用户不会某天收到精美 AI 文案、第二天收到生硬模板。

### 5.4 小鸟跨天记忆

当前设计里小鸟只记得“昨天来没来”，这不足以支撑持续陪伴的情感需求。真正让用户回来的，是“它还记得我昨天说过的那句话”。

V0.2 做最轻量的跨天记忆系统：

**存储**：`bird_memories` 表，每天聊天结束后异步生成一条记忆摘要（最多保留最近 7 天）。

```
字段：user_id, pet_id, date_key, memory_text, memory_type
```

**使用**：次日进入聊天页时，随机抽取 1 条记忆注入到小鸟的开场白 prompt 中：

```
你昨天说过：“{memory_text}”
今天如果自然的话，可以在开场或聊天中轻轻提一下。
不要刻意，不要像任务。就像朋友不经意想起一样。
```

**示例效果**：

| 昨天的对话 | 今天的开场白 |
|---|---|
| 用户说工作很不开心 | “昨天你说完那些以后，我在窗边看到一片特别亮的叶子。想给你看。” |
| 用户分享了一件开心事 | “你昨天说的那件事，我今天想起来还是替你高兴。” |
| 用户只是随便聊了几句 | （不主动提起，保持自然） |

**规则**：

1. 记忆不强制使用——如果上下文不自然，小鸟不提。
2. 用户感知不到“系统在调取记忆”，只觉得小鸟很贴心。
3. 每段记忆 7 天后自动清除，不给系统负担也不给用户隐私压力。
4. 记忆只对当前小鸟生效，串门朋友的小鸟看不到。

### 5.5 小鸟的行为特征（鸟的天性）

一只鸟不只是会说话。它应该有一些**鸟独有的小行为**，这些行为是猫、兔子、植物做不到的。

V0.2 至少做一个鸟的特有行为：

| 行为 | 说明 | 实现方式 | V0.2 是否做 |
|---|---|---|---|
| **叼东西回来** | 小鸟偶尔叼来一片叶子/花瓣/亮片/小石头，放在首页“小鸟的收藏”里 | 随机事件 + 首页轻展示 | [必做] 见 4.3 |
| **对天气有反应** | 下雨天小鸟缩在窝里，晴天在窗边晒太阳 | 调用天气 API + mood_text 变化 | [建议做] |
| **唱歌/鸣叫** | 每天给你哼一段不同的旋律 | 可播放短音频 | V0.3 |
| **整理羽毛** | 小鸟偶尔有整理羽毛的动画/状态描述 | 状态文案 + 动画 | V0.3 |
| **认人** | 对串门的朋友有不同反应（“有陌生的小鸟来了”vs“xx 的小鸟又来了”） | 串门文案差异化 | [建议做] |
| **飞行/迁徙** | 小鸟在虚拟地图上飞来飞去，偶尔发现新地方 | 虚拟探索系统 | V0.3+ |

最低要求：V0.2 必须让用户感受到“这是一只鸟，不是一个换了个头像的 chatbot”。至少做到叼东西 + 天气反应。

## 6. 页面结构

### 6.1 首页：小鸟主屏

首页要从“信息卡”变成“小鸟今天的状态”。

信息层级：

1. 小鸟形象。
2. 小鸟名字。
3. 今日心情标签。
4. 小鸟今日一句话。
5. 今日鸟签入口。
6. 和小鸟说话入口。
7. 今日完成反馈。
8. 串门提醒。

建议结构：

```text
顶部：品牌/轻导航
主体：大鸟图 + 今日心情
中部：小鸟一句话
行动区：今日鸟签 / 和小鸟说话
底部：羽毛、熟悉度、串门提醒
```

首页示例文案：

```text
它今天有点困困。
脑袋埋在羽毛里，但听见你来了，还是抬头看了一眼。
```

按钮：

```text
看今天的小鸟签
陪它说几句
```

### 6.2 今日鸟签页

鸟签页要更有仪式感，不只是展示文字。

页面流程：

1. 未查看：展示“小鸟叼来一张签”。
2. 点击打开。
3. 展示鸟签标题、正文、行动建议。
4. 展示小鸟反应。
5. 返回首页后首页状态更新。

小鸟反应示例：

```text
它看了看这张签，又看了看你。
“今天可以慢一点。”
```

### 6.3 聊天页

聊天页要体现“小鸟在陪你”，不是普通 AI 助手。

新增要点：

1. 进入聊天页时，小鸟主动开场。
2. 发送后出现“小鸟正在想...”。
3. 回复逐字出现。
4. 聊天完成后影响心情。
5. 首次聊天完成后提示“今天它更安心了一点”。

开场白按 mood 变化：

| mood | 开场白 |
|---|---|
| sleepy | 我有点困，但可以陪你说一会儿。 |
| bright | 我刚刚想到你。今天想说点什么？ |
| quiet | 我在这里。你可以慢慢说。 |
| waiting | 你来了。我等了一小会儿。 |
| messy | 我的羽毛也有点乱，我们慢慢理。 |
| cozy | 靠近一点说也可以。 |

### 6.4 小鸟设定页

继续保留昵称和人设 prompt，但语气要更产品化。

字段：

1. 小鸟名字。
2. 小鸟说话方式。
3. 小鸟不该怎么说。

示例：

```text
希望它怎么陪你？
比如：说话短一点，先安慰我，不要讲大道理。
```

### 6.5 串门提醒

串门暂时不做复杂社交，作为轻惊喜。

展示位置：首页下方。

示例：

```text
朋友的小鸟来窗台停了一会儿。
它带回来一句话：今天也有人在想着你。
```

**回访机制**：串门不能是单向死胡同。当 A 看到 B 的小鸟来过时，可以点击“让我的小鸟也去看看它”，触发一次反向串门。B 次日进入时能看到 A 的小鸟回访的提示。这样形成最简单的互访闭环，不需要私信、不需要即时通讯。

回访规则：

1. A 收到 B 的串门通知后，首页出现“去看看它？”按钮。
2. 点击后 A 的小鸟“飞过去”，B 次日收到回访通知。
3. 每对朋友每天最多互访一次，防止刷屏。
4. 回访不产生新内容，只产生一个温柔的通知：“{A 的小鸟名} 昨天来看过你的小鸟。它在窗台上留了一片小叶子。”

用途：用户真正打开今日鸟签。

3. 标记 `todayCare.sign_opened = true`。

原有接口保留。

3. 可能更新 mood。
4. 如果 sign 和 chat 都完成，生成今日小纸条。

## 9. AI Prompt 方向

### 9.1 聊天 system prompt

重点：

1. 它是一只小鸟，不是通用 AI 助手。
2. 回复短、柔软、具体。
3. 先接住情绪，再给一个小动作。
4. 不做诊断，不讲大道理。
5. 根据 mood 和 bond_level 调整亲密度。

### 9.2 小纸条生成 prompt

输入：

1. 今日鸟签。
2. 小鸟 mood。
3. 用户最后一句聊天。
4. 小鸟 persona_prompt。

输出：

1. 一句话。
2. 40 字以内。
3. 像小鸟留给用户的小纸条。
4. 不要鸡汤，不要命令。

## 12. UI 高保真方向

视觉关键词：

```text
温柔、轻、奶油色、纸感、真实鸟图、JOYI 红棕、深绿文字、像一张每天打开的小鸟便签
```

避免：

1. 过度游戏化。
2. 大量积分、等级、任务感。
3. 太像心理咨询 App。
4. 太像宠物游戏。
5. 信息卡堆叠。

首页高保真方向：

1. 第一屏必须看到大鸟图。
2. 文案少，情绪明确。
3. 行动入口最多两个主按钮。
4. 熟悉度和羽毛作为轻信息，不抢主流程。
5. 底部 tab 更轻、更像小程序常用导航。

## 13. 风险与边界

### 13.1 风险

1. 玩法过重会破坏轻陪伴气质。
2. 宠物状态如果有惩罚，会让用户有负担。
3. AI 回复不稳定会影响小鸟人格。
4. 4 位兑换码简单，内测可用，正式版要加强风控。

### 13.2 边界

V0.2 不做：

1. 商城。
2. 复杂鸟窝装修。
3. 多宠物。
4. 排行榜。
5. 陌生人社交。
6. 私信。
7. 复杂任务树。
8. 付费订阅。

## 14. 推荐推进顺序

1. 先做 Slice 1：首页小鸟状态体系。
2. 再做 Slice 2：鸟签仪式感。
3. 再做 Slice 3：聊天影响小鸟。
4. 再做 Slice 4：今日小纸条。
5. 最后做 Slice 5：串门惊喜。

第一步不要先画全量 UI。先把首页状态和数据跑通，再高保真。

## 15. V0.2 第一周验收标准

1. 用户打开首页时，能感受到“小鸟今天有状态”，并且能看到小鸟今天发生了什么（叙事层）。
2. 用户查看鸟签后，首页反馈发生变化。
3. 用户聊天后，小鸟心情或熟悉度发生变化。
4. 用户完成今日陪伴后，收到一句小纸条（模板生成，AI 可选润色）。
5. 后台能看到 mood、bond、feather、care 记录。
6. **（新增）** 小鸟至少有一个鸟独有的行为（如叼东西、天气反应），让用户感觉“这是一只鸟”。
7. **（新增）** 小鸟有跨天记忆：在聊天中能自然提起用户昨天说过的事。
8. **（新增）** 每天有低概率触发随机事件（15%-25%），给用户带来意外感。
9. 整体体验比 V0.1 更可爱，但不增加明显操作负担。
