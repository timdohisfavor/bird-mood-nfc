# JOYIBIRD 小程序 V0.2 宠物设计系统

## 1. 设计目标

V0.2 的小鸟不再沿用 H5 的真实鸟类插画逻辑，而是升级为用户可拥有、可期待、可养成、可装扮的游戏宠物。

核心变化：

- 从“抽到一只鸟”变成“拥有一只小鸟伙伴”。
- 从“鸟类资料展示”变成“宠物状态、情绪和互动反馈”。
- 从“单张图片”变成“可扩展的宠物资产系统”。
- 从“鸟签工具”变成“每天回来看看小鸟”的轻养成体验。

## 2. 视觉母题

参考圆鸟系列 NFC 手机磁吸气囊支架的全家福，保留以下品牌资产：

- 圆形徽章感。
- 奶油色背景。
- JOYI 红棕色品牌标识。
- 6 只小鸟的强色彩区分。
- 收藏玩具、徽章、挂件、盲盒的期待感。

小程序宠物应比原产品图更拟人、更软、更像游戏角色。

推荐基础风格：

- 3D plush toy / soft vinyl toy。
- 圆润、短腿、胖肚子、大眼睛。
- 羽毛细节减少，保留关键识别色块。
- 头部和身体比例偏可爱化，约 1:1 或 1.2:1。
- 像可以摆在手机壳、钥匙扣、徽章里的小圆鸟。

避免：

- 写实鸟类图鉴。
- 过细羽毛纹理。
- 过成人化、过高冷。
- 单纯把真实鸟图套进 UI。
- 表情不可读或动作太小。

## 3. 六只默认宠物

### 3.1 北长尾山雀

定位：云朵小团子。

视觉识别：

- 主色：白、浅灰、粉褐。
- 体型：最圆、最轻、像一颗棉花球。
- 尾巴：细长向上翘，是主要剪影特征。
- 表情：小小黑豆眼，安静但黏人。

性格：

- 害羞、温柔、喜欢贴贴。
- 用户情绪低落时最适合陪伴。

可爱动作：

- 把头埋进胸前羽毛。
- 小跳两下再看用户。
- 尾巴轻轻摇。

装饰适配：

- 小围巾、绒球帽、云朵背饰、热牛奶杯。

### 3.2 普通翠鸟

定位：小小冒险家。

视觉识别：

- 主色：蓝绿色、橙黄色、浅蓝。
- 体型：圆身体配长小嘴。
- 特征：高饱和蓝绿背部和橙色肚子。
- 表情：眼神亮，像发现了什么。

性格：

- 好奇、行动力强、爱探索。
- 适合带出“今天试试看”的行动建议。

可爱动作：

- 叼起一颗小水珠。
- 歪头观察。
- 像小火箭一样短短冲一下。

装饰适配：

- 小护目镜、探险帽、鱼形挂件、蓝色水波底座。

### 3.3 伯劳

定位：外冷内软的小守卫。

视觉识别：

- 主色：米白、暖棕、黑色眼罩。
- 体型：胸脯挺一点，轮廓更有精神。
- 特征：黑色眼罩、暖棕翅膀、较利落的小嘴。

性格：

- 嘴硬、可靠、保护欲强。
- 可以给用户一点轻轻的边界感和鼓励。

可爱动作：

- 假装严肃地站岗。
- 发现用户看它后悄悄脸红。
- 用翅膀轻轻挡在用户前面。

装饰适配：

- 小披风、护目镜、小旗子、守卫徽章。

### 3.4 红尾水鸲

定位：蓝色小火苗。

视觉识别：

- 主色：蓝、深蓝、橙红尾巴。
- 体型：圆但更活泼，尾巴颜色强。
- 特征：红橙色尾巴像小火苗。

性格：

- 活泼、热心、容易兴奋。
- 适合做每日任务、互动奖励的引导者。

可爱动作：

- 尾巴像小火苗一样抖动。
- 开心时转一圈。
- 把用户的坏心情“烧掉一点点”。

装饰适配：

- 运动发带、小鼓、火苗尾饰、星星贴纸。

### 3.5 红头长尾山雀

定位：元气小麻薯。

视觉识别：

- 主色：红褐、灰、奶白、黑色小脸纹。
- 体型：圆润但比北长尾山雀更有精神。
- 特征：红头、黑色脸部花纹、长尾。

性格：

- 热情、黏人、话多一点。
- 适合主动和用户打招呼。

可爱动作：

- 一边蹦一边叫用户。
- 把小脸凑近屏幕。
- 开心时尾巴画小弧线。

装饰适配：

- 发夹、红色小斗篷、贴纸包、小铃铛。

### 3.6 红胁秀眼

定位：青柠软糖。

视觉识别：

- 主色：柠檬绿、黄绿、白眼圈、浅橙侧腹。
- 体型：小巧圆润，颜色最明亮。
- 特征：白眼圈必须保留。

性格：

- 清爽、机灵、轻快。
- 适合提醒用户喝水、休息、做一件小事。

可爱动作：

- 眨眼圈。
- 轻轻啄一下小叶子。
- 开心时身体像软糖一样弹一下。

装饰适配：

- 小叶子帽、透明雨披、果冻背包、青柠徽章。

## 4. 宠物动作矩阵

V0.2 宠物不再是静态图鉴卡片，而是随玩法切换状态和表情的陪伴角色。下面定义 11 个标准动作态，覆盖首页待机、聊天交互、每日仪式、随机事件和天气/记忆联动。

### 4.1 动作态总览

| 动作 key | 中文名 | 触发场景 | 出现频率 |
|----------|--------|---------|---------|
| `idle` | 待机 | 首页默认 | 每天 |
| `happy` | 开心 | 完成互动 / 收到鸟签后 | 每天 |
| `talking` | 聊天中 | AI 对话回复时 | 每天 |
| `sleepy` | 困困 | 夜间或久未互动 | 时段性 |
| `waiting` | 等待 | 等待用户输入或点击 | 每天 |
| `reward` | 获得礼物 | 收到羽毛/装饰/串门礼物 | 间歇 |
| `waking` | 醒来 | 每日首次打开 | 每天 1 次 |
| `carrying` | 叼东西 | 随机事件（15%-25% 概率） | 4-7 天 1 次 |
| `shy` | 害羞 | 亲密度低时打招呼 | 初期常见 |
| `rainy` | 天气反应 | 当日天气联动 | 按天气 |
| `miss` | 想念 | 跨天记忆：昨天没来 | 偶发 |

### 4.4 生图规格与优先顺序

#### 规格要求
- 格式：PNG，720×720，透明底
- 风格：3D plush toy / soft vinyl toy，圆润短腿胖肚子
- 背景：无背景（透明），无树枝/叶子/场景元素
- 画面：独立鸟体，保留物种关键识别色块和特征

#### 优先顺序

| 优先级 | 动作 | 每只鸟都做 | 原因 |
|--------|------|-----------|------|
| 🔴 P0 | `idle` | ✅ 已有 | 首页基础态 |
| 🔴 P0 | `happy`, `talking`, `sleepy`, `waiting`, `reward` | ✅ | 覆盖首页切换 + 聊天交互 + 每日闭环 |
| 🟡 P1 | `waking`, `carrying`, `shy` | ✅ | 每日唤醒仪式 + 随机惊喜 + 初期亲密度 |
| 🟢 P2 | `rainy`, `miss` | ✅ | 天气联动 + 跨天记忆（V0.2 后期） |

首轮（Slice 0）建议每只先做 P0 的 6 态，再分批补 P1/P2。

## 5. 装饰系统

装饰是后续增值服务之一，所以宠物本体设计时必须预留挂载位置。

装饰槽位：

- `head`：帽子、发夹、叶子、小皇冠。
- `neck`：围巾、项圈、小铃铛。
- `back`：小背包、披风、翅膀贴饰。
- `badge`：圆环徽章、纪念贴纸、任务章。
- `room`：鸟窝背景、天气、桌面摆件。

装饰原则：

- 不遮挡鸟种关键特征。
- 不改变宠物身份，只改变情绪和收藏感。
- 稀有装饰可以带轻微动画，但 V0.2 先用静态资源。

## 6. 首页体验方向

首页应像一个小型宠物房间，而不是信息卡片。

首屏结构：

1. 顶部：今日问候和小鸟心情。
2. 中心：大尺寸宠物，带轻微待机动画。
3. 宠物旁：心情气泡或一句主动问候。
4. 下方：今日鸟签、和小鸟说话、装扮入口。
5. 角落：亲密度、连续陪伴天数、串门提醒。

首页核心反馈：

- 用户每天第一次打开，小鸟醒来。
- 看完鸟签，小鸟变开心或若有所思。
- 聊天后，亲密度增加一点。
- 分享后，小鸟期待朋友来串门。

## 7. 第一版概念图要求

生成首版概念图时，目标不是最终资产，而是判断方向。

画面要求：

- 一张 2x3 六只宠物设定图。
- 奶油色背景，延续 JOYI 品牌。
- 每只鸟都有圆形徽章或圆形底座，但宠物本体更立体、更软萌。
- 不出现文字，不出现真实 logo。
- 每只鸟要保留物种关键识别点。
- 风格统一，像同一个游戏里的默认宠物系列。

首版关键词：

soft vinyl toy, plush mascot, round bird pet, collectible badge, mobile game pet, cozy, cute, high quality, cream background.
